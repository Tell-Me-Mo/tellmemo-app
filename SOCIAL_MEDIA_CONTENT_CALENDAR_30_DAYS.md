### Day 27: Technical Deep Dive - RAG Architecture

**LinkedIn Post:**
```
Let's get technical: How TellMeMo's RAG system actually works üîß

For the engineers and AI enthusiasts:

**Problem:** Generic LLMs hallucinate. They confidently make up facts.

**Solution:** RAG (Retrieval Augmented Generation)

**TellMeMo's RAG Pipeline:**

**STEP 1: Content Ingestion**
```
Meeting transcript uploaded
‚Üí Validated (format, size, content)
‚Üí Stored in PostgreSQL (metadata)
‚Üí Queued for processing (Redis Queue, high priority)
```

**STEP 2: Chunking**
```
Full transcript ‚Üí Text splitter
‚Üí ~500-word chunks (overlap: 50 words)
‚Üí Preserve context (speaker, timestamps)
‚Üí Metadata attached (meeting_id, project_id, org_id)
```

**STEP 3: Embedding Generation**
```
Each chunk ‚Üí EmbeddingGemma-300m (local)
‚Üí 768-dimensional vector
‚Üí Batch processing (efficiency)
‚Üí Store in Qdrant (organization collection)
```

**STEP 4: Vector Storage**
```
Qdrant point structure:
{
  "id": "content_uuid_chunk_0",
  "vector": [768 floats],
  "payload": {
    "content": "chunk text",
    "metadata": {...},
    "project_id": "...",
    "organization_id": "..."
  }
}
```

**STEP 5: Query Processing**
```
User asks: "What risks were identified?"
‚Üí Query ‚Üí EmbeddingGemma ‚Üí 768-dim vector
‚Üí Qdrant semantic search (org + project filtered)
‚Üí Cosine similarity >0.75
‚Üí Top 10 chunks retrieved
```

**STEP 6: Context Assembly**
```
Retrieved chunks ‚Üí
Sort by relevance ‚Üí
Deduplicate by source ‚Üí
Construct prompt:
  "Context: [chunks]
   Question: [user query]
   Instructions: Answer using only the context..."
```

**STEP 7: LLM Generation**
```
Prompt ‚Üí Claude 3.5 Haiku
‚Üí Streaming response
‚Üí Extract source citations
‚Üí Format markdown
‚Üí Return to user
```

**STEP 8: Fallback Handling**
```
Claude overloaded (529 error) ‚Üí
Circuit breaker pattern ‚Üí
‚Üí Automatic OpenAI GPT-4o fallback
‚Üí Zero user disruption
‚Üí Langfuse tracks both providers
```

**Key Technical Decisions:**

**1. Why local embeddings (EmbeddingGemma)?**
‚úÖ Zero API costs
‚úÖ Fast inference (CPU-optimized)
‚úÖ Privacy (no external calls)
‚úÖ Sufficient quality (768-dim)

**2. Why Qdrant over Pinecone/Weaviate?**
‚úÖ Self-hostable
‚úÖ Excellent filtering (org isolation)
‚úÖ Fast (HNSW index)
‚úÖ Active development

**3. Why Claude over GPT?**
‚úÖ Superior reasoning
‚úÖ 200K context window
‚úÖ Better at following instructions
‚úÖ + GPT fallback for HA

**4. Why Redis Queue over Celery?**
‚úÖ Simpler setup
‚úÖ Great dashboard (RQ Dashboard)
‚úÖ Python-native
‚úÖ Sufficient for current scale

**Performance Metrics (Production):**

‚Ä¢ Embedding generation: ~50ms/chunk
‚Ä¢ Semantic search: <100ms (p95)
‚Ä¢ End-to-end query: 3-6 seconds
‚Ä¢ Concurrent users: 50+ (tested)
‚Ä¢ Vector DB size: Scales linearly
```
